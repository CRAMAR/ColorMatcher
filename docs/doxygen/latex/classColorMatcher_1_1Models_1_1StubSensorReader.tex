\doxysubsection{Detailed Description}
Mock/stub implementation of ISensor\+Reader for development and testing without physical hardware. 

This implementation simulates a realistic spectrophotometer (NIX Mini 3 model) by\+:
\begin{DoxyItemize}
\item Generating plausible simulated color readings
\item Supporting quality score variations to test validation logic
\item Allowing manual override for deterministic testing
\item Maintaining session history of readings
\item Simulating realistic measurement patterns and delays
\end{DoxyItemize}

{\bfseries{Use Cases}}


\begin{DoxyEnumerate}
\item {\bfseries{Development}}\+: Develop and test color matching UI without physical hardware
\item {\bfseries{Unit Testing}}\+: Mock sensor behavior for View\+Model and business logic tests
\item {\bfseries{Demo/\+Prototype}}\+: Quickly prototype sensor integration without hardware setup
\item {\bfseries{CI/\+CD}}\+: Run automated tests in build pipelines without sensor dependencies
\end{DoxyEnumerate}

{\bfseries{Realistic Simulation Features}}


\begin{DoxyItemize}
\item {\bfseries{Color Clustering}}\+: Generates colors grouped around common paint colors (red, green, blue, gray) rather than purely random colors, matching typical color matching use cases
\item {\bfseries{Quality Score Distribution}}\+: 80\% "{}good"{} (75-\/100), 20\% "{}fair"{} (50-\/75) to simulate realistic environmental variations (dust, lighting, angle)
\item {\bfseries{Async Behavior}}\+: All methods are truly async (return Tasks) to match real sensor behavior
\item {\bfseries{Device Metadata}}\+: Reports realistic device name, ID, and status simulating a real sensor
\item {\bfseries{Session Tracking}}\+: Maintains reading history and count for audit trails
\end{DoxyItemize}

{\bfseries{Manual Override for Testing}}

Use Set\+Next\+Reading\+Override() to force specific colors for deterministic tests\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ sensor\ =\ \textcolor{keyword}{new}\ StubSensorReader();}
\DoxyCodeLine{await\ sensor.ConnectAsync();}
\DoxyCodeLine{sensor.SetNextReadingOverride(\textcolor{keyword}{new}\ RgbColor(255,\ 0,\ 0));\ \textcolor{comment}{//\ Force\ red}}
\DoxyCodeLine{var\ reading\ =\ await\ sensor.ReadColorAsync();\ \textcolor{comment}{//\ Returns\ red}}
\DoxyCodeLine{\textcolor{comment}{//\ reading.RgbColor\ will\ be\ (255,\ 0,\ 0)}}

\end{DoxyCode}


After returning the overridden color, reverts to random generation for subsequent calls.

{\bfseries{Limitations}}


\begin{DoxyItemize}
\item Does not simulate hardware communication failures or timeouts
\item Calibrate\+Async() always succeeds (doesn\textquotesingle{}t simulate calibration issues)
\item Doesn\textquotesingle{}t model sensor drift or measurement variation over time
\item Battery simulation is hardcoded (always \texorpdfstring{$\sim$}{\string~}85\%)
\end{DoxyItemize}

These limitations are acceptable for development and testing purposes. For production, real sensor implementations should handle failure scenarios.

Returns a validated color reading with automatic retry and quality checking. 

Process\+:
\begin{DoxyEnumerate}
\item Loop up to max\+Retries times
\item Call Read\+Color\+Async() to get measurement
\item Check if Quality\+Score \texorpdfstring{$>$}{>}= minimum\+Quality\+Score
\item If valid, return immediately
\item If not valid and retries remain, pause 100ms and try again
\item Return last reading (even if below quality threshold)
\end{DoxyEnumerate}

This ensures the application always gets a reading, even if quality is suboptimal.
